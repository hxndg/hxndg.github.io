#### 1.2.1 CACHE的工作原理

一个cache，由多个cache block组成。单个cache block由tag字段，状态位和数据单元组成。status包含MESI，MOESI或者MESIF的状态信息

#### 1.2.2 CACHE的组成结构

+ Sector Buffer结构
+ Set Associate结构：包括Direct Mapped和Fully-Associate结构（这两个是特例）



当S参数为1的时候，这个就是Fully-Associate。当N参数为1的时候，管理方式为Directed Mapped



### 1.3 第三章conherence and consistent笔记

根据 Cache Coherency Protocol 维护 Cache Block 状态方法的不同，处理器可以使用 Bus Snooping 和 Directory 这两大类机制。这两类机制的主要区别在于 Directory 机制全局统一管 理不同 Cache 的状态;而在 Bus Snooping 机制中，每个 Cache 分别管理自身 Cache Block 的 状态，并通过 Interconnection 进行不同 Cache 间的状态同步。

有多种一致性协议：

+ Write-Once Protocol 的实现关键在于其使用的特殊的 Cache 回写机制，即 Write Once。 Write-Once 是 Write-Back 和 Write-Through 的综合。当使用这种机制时，对一个 Cache Block 进行第一次回写时，采用 Write-Through 策略将数据同时回写到 Cache 和主存储器，之后的 写操作采用 Write-Back，只回写到 Cache 而不回写到主存储器。
