今天的目标就是将redis-plus-plus库（c++库）从cmake编译转换为bazel编译，redis-plus-plus依赖hiredis库，hiredis库使用源码引入，redis-plus-plus依赖的uv库我们使用外部引入。

包含三个部分：

如何使用bazel编译没有SSL支持的sync & async 的hiredis

如何使用bazel编译第三方外部依赖（非bazel项目）openssl

如何使用bazel编译第三方外部依赖（非bazel项目）libuv

如何使用bazel翻译redis++库从cmake到bazel的转换，包含sync & async & ssl part



## 1 hiredis部分

在基础目录



无论是bazel还是cmake，实际上都在追究一个头文件可见性。一些老的c/c++代码是直接把头文件和代码放到一起，然后直接`include "foo.h"`。cmake通过指定TARGET_INCLUDE_DIRECTORIES来确定引用哪些目录，然后调用者可以类似上面`include "foo.h"`调用头文件。这里面引发一个问题，直接`target_link_libraries`，头文件会使用库引入而直接暴露的头文件，也就是上一层库暴露的头文件，参考链接：https://ravenxrz.ink/archives/e40194d1.html，而bazel是采用相对于workspace的相对路径来引用具体的文件。

```cmake
CMAKE_MINIMUM_REQUIRED(VERSION 3.4.0)
INCLUDE(GNUInstallDirs)
PROJECT(hiredis)

OPTION(ENABLE_SSL "Build hiredis_ssl for SSL support" OFF)

MACRO(getVersionBit name)
  SET(VERSION_REGEX "^#define ${name} (.+)$")
  FILE(STRINGS "${CMAKE_CURRENT_SOURCE_DIR}/hiredis.h"
    VERSION_BIT REGEX ${VERSION_REGEX})
  STRING(REGEX REPLACE ${VERSION_REGEX} "\\1" ${name} "${VERSION_BIT}")
ENDMACRO(getVersionBit)

getVersionBit(HIREDIS_MINOR)

        DESTINATION ${CMAKE_INSTALL_LIBDIR}/pkgconfig)

    export(EXPORT hiredis_ssl-targets
           FILE "${CMAKE_CURRENT_BINARY_DIR}/hiredis_ssl-targets.cmake"
           NAMESPACE hiredis::)

                                  PATH_VARS INCLUDE_INSTALL_DIR)

    INSTALL(EXPORT hiredis_ssl-targets
        FILE hiredis_ssl-targets.cmake
        NAMESPACE hiredis::
        DESTINATION ${CMAKE_CONF_INSTALL_DIR})

    INSTALL(FILES ${CMAKE_CURRENT_BINARY_DIR}/hiredis_ssl-config.cmake
        DESTINATION ${CMAKE_CONF_INSTALL_DIR})
ENDIF()

IF(NOT DISABLE_TESTS)
```

**CFLAGS：** 指定头文件（.h文件）的路径，如：CFLAGS=-I/usr/include -I/path/include。同样地，安装一个包时会在安装路径下建立一个include目录，当安装过程中出现问题时，试着把以前安装的包的include目录加入到该变量中来。可以看到下面制定了具体安装的路径是默认的prefix/include/hiredis。当然因为我们不适用pkg软件了，所以实际上我们不需要关注这个。

```makefile
prefix=@CMAKE_INSTALL_PREFIX@
install_libdir=@CMAKE_INSTALL_LIBDIR@
exec_prefix=${prefix}
libdir=${exec_prefix}/${install_libdir}
includedir=${prefix}/include
pkgincludedir=${includedir}/hiredis

Name: hiredis
Description: Minimalistic C client library for Redis.
Version: @PROJECT_VERSION@
Libs: -L${libdir} -lhiredis
Cflags: -I${pkgincludedir} -D_FILE_OFFSET_BITS=64
```

```BUILD
cc_library(
    name = "hiredis",
    srcs = [
        "alloc.c",
        "dict.c",
        "async.c",
        "hiredis.c",
        "net.c",
        "read.c",
        "sds.c",

    hdrs = glob(["*.h"])+["dict.c",],

    name = "hiredis_ssl",
    srcs = [
```
